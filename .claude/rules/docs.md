# Documentation Conventions

- Research docs: `docs/research/YYYY-MM-DD-slug.md` with date of research
- ADRs: `docs/decisions/ADR-NNN-slug.md` — sequential, never delete, never renumber
- Design docs: `docs/design/slug.md` — implementation specifications
- Hardware docs: `docs/hardware/` — LOCKED inventory, audits, specs
- All claims cite sources (URLs, docs, benchmarks, release notes)
- Tables for structured data (specs, comparisons, status)
- Code blocks for configs and commands
- No fluff, no filler, no marketing language
- Cross-reference related docs with relative links
- Include "Last updated: YYYY-MM-DD" footer when modifying
- Hardware inventory changes require Shaun's explicit confirmation
- Research must compare minimum 3 options with quantitative data
