## Rules

- Read routes must not silently mutate operator or platform state.
- Privileged writes must require operator session proof and upstream service auth.
- Shared operator state belongs in server-backed stores, not local storage.
- Terminal access is treated as a privileged operator capability, not a public dashboard widget.
- Signed terminal bridge tickets must preserve operator identity for downstream audit correlation, and bridge accept/deny decisions must emit best-effort operator audit events.
- Transitional helper routes such as `/api/agents/proxy`, `/api/containers/[name]/restart`, and `/api/gpu/swap` must stay operator-gated until they are fully retired behind canonical upstream contracts.
